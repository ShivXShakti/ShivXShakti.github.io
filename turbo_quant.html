Where do these variables come from?

In a Transformer network (like Llama-3), the attention mechanism is the core. TurboQuant operates on the KV Cache—the memory of past tokens.

Attention Matrix Equation
\[ \text{Attention}(Q, K, V) = \text{Softmax}\left( \frac{Q \times K^\top}{\sqrt{d}} \right) \times V \]

Data Flow: From Transformer Layer to KV Cache

Variable Legend (From Network)

Q, K, V Projections: Result of \(h_t \times W\). \(K \in \mathbb{R}^{1 \times d}\) and \(V \in \mathbb{R}^{1 \times d}\) are what we compress. Traditional Quantization & The Role of G

Standard quantization splits the d=128 vector into smaller Groups of size G to maintain precision. However, every group carries a "Metadata Tax".

Range-Based Scaling Equations
1. Scale: \( s = \frac{x_{max} - x_{min}}{2^b - 1} \) 2. Quant: \( q(x) = \text{Round}\left( \frac{x - x_{min}}{s} \right) \) 3. Dequant: \( \hat{x} = q(x) \times s + x_{min} \)

The "Metadata Tax" Problem (d=128, G=8)

Variables in Traditional Quantization

PolarQuant: Geometric Efficiency & Spherical Math

PolarQuant replaces "Grid Math" (Cartesian) with "Circle Math" (Polar). After rotation, the range of angles is fixed and known, eliminating the need for per-group metadata.

Conceptual Shift: Cartesian vs. Polar

Spherical Coordinate Math Worked Example (2D Pair)

Imagine a pair of values from a Key vector projection: \(x = 0.707, y = 0.707\).

Step 1: Calculate the Radius (\(r\)) Find the magnitude using the Pythagorean theorem: \[ r = \sqrt{x^2 + y^2} = \sqrt{0.707^2 + 0.707^2} = \sqrt{0.5 + 0.5} = \mathbf{1.0} \] Step 2: Calculate the Angle (\(\theta\)) Find the direction using atan2: \[ \theta = \text{atan2}(y, x) = \text{atan2}(0.707, 0.707) = \pi/4 \text{ radians } (45^\circ) \] Step 3: Quantize the Angle Since \(\theta \sim \text{Uniform}[-\pi, \pi)\), we divide the circle into \(2^b\) even slices (e.g., 16 slices for 4-bit). The angle is stored as a slice index. No scale or min is needed!
Why This Saves Memory

Rotation Scrambles the data so angles become statistically uniform and predictable. By storing only the final radius, we escape the "tax".

QJL: 1-Bit Error Correction

PolarQuant is great, but rounding introduces error (\(\epsilon\)). QJL uses a tiny 1-bit check to ensure the result is unbiased.

Stochastic Error Estimation
1. Error: \( \epsilon = k' - \tilde{k} \) 2. Correction Bits: \( c = \text{Sign}( R_{JL} \times \epsilon ) \) 3. Unbiased Estimate: \( \langle q, k \rangle \approx \langle q, \tilde{k} \rangle + \frac{\pi}{2m} c^\top \times (R_{JL} \times q) \)

The 1-Bit Residual Correction

QJL Variables
ε Residual: The info lost from PolarQuant (\( k' - \tilde{k} \in \mathbb{R}^d \)). R_JL JL Matrix: A random projection matrix (\( R_{JL} \in \mathbb{R}^{m \times d} \)). c Correction Bits: The sign bits \( c \in \{-1, 1\}^m \). m Bit Budget: Number of correction bits (e.g., 32 bits).
